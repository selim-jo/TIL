# 스프링 배치
- 배치란? 데이터를 원하는 대로 가공해서 처리하는 자동화된 프로세스
- 배치는 일반적으로 읽기->처리->쓰기의 3단계로 이루어짐
- 배치(batch) = 일괄 처리 = 큰 데이터를 한번에 처리하고 결과를 저장 = 특정 주기로 자동으로 실행되는 프로그램임
- 사용 예시
: 회원 탈퇴 시, 해당 회원과 관련된 데이터를 다 삭제해야한다고 할 때, 갑자기 많은 탈퇴자가 있을시 많은 양의 데이터를 처리하기 어려움
: so, 사람들이 많이 몰리지 않는 새벽 시간에 자동화 처리를 해놓고 그때 처리함
- 배치 서비스 특징
1) 대용량 데이터
2) 자동화
3) 견고성
- 또한, 추가적으로 배치는 실무에서 많은 것들을 편하게 해줌
- 실시간으로 특정 데이터를 처리했을 때, 해당 처리 결과가 제대로 처리되지 않았다면?
-> 배치로 실패한 결과물에 대해서 재실행을 실시하여 커버 가능
- 배치는 실제 로그를 쌓을 수 있으므로, 결과에 대한 확인이 가능해짐 

# JNDI(Java Naming and Directory Interface)
- 디렉터리 서비스에서 제공하는 데이터 및 객체를 발견하고 참고하기 위한 자바 API
- JNDI 용도
1) 자바 애플리케이션을 외부 디렉터리 서비스에 연결 (ex) 주소 데이터베이스 또는 LDAP 서버)
2) 자바 애플릿이 호스팅 웹 컨테이너가 제공하는 구성 정보를 참고
- 즉, 우리가 연결하고 싶은 데이터베이스의 DB Pool을 미리 Naming을 시켜주는 방법 중 하나임
  우리가 저장해놓은 WAS의 데이터베이스 정보에 JNDI를 설정해 놓으면 웹 애플리케이션은 JNDI만 호출하면 간단해짐
- 그럼 이것을 왜 사용하는가?
-> 우리는 보통 JDBC를 설정해서 개발을 함. 하지만 웹 애플리케이션을 운영서버로 만들 경우 얘기가 달라짐
-> So, 운영/관리/최적화 문제 대처에 다양한 이점이 있기 때문에 JNDI 사용함
