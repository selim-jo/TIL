# Redis 서버
- 이전에 Node.js 프로젝트를 하면서 채팅 기능을 구현했었는데 
  채팅 같은 경우 쌓인 데이터 조회가 많다보니 서비스 요청이 증가하면 데이터를 조회해올 때 많은 부하가 일어날 수 있을 것 같아 Redis를 적용하려 했었다.
  But, 마지막에 시간이 부족해서 적용하지 못했지만...
  그렇게 잊고 있었는데, Redis를 접하게 되어 개념을 정리하고 어떻게 사용하는지에 대해 정리하려고 한다.
- Redis란?
: In-memory 기반의 NoSQL DBMS로서 Key-Value의 구조를 가지고 있으며, 속도가 빠르고 사용이 간편함
: 보통 캐싱/세션관리 등으로 쓰임
- SpringBoot와 Redis 연동하는 법은 공부하면서 연동해봐야겠다.
