# 자바 기초 개념 정리
- 용어 정리
1) 데이터: 컴퓨터가 정보를 담는 자료값
2) 데이터 타입(자료형): 데이터의 유형 ex) 정수/문자 등등
3) 자료구조: 데이터 여러개를 분류해서 담을 수 있는 분류통
4) 문맥: 개발언어로 표현한 프로그램의 동작 흐름을 의미함
5) 메서드: 데이터와 문맥을 감싸는 포장지
6) 공통 실행환경: 여러 기기의 운영환경에서 실행할 수 있도록 판을 깔아주는 실행환경
7) 운영 체제: 기기가 프로그램을 실행시키는 운영환경 ex) Window, Android, iOS, Linux...

- Java 언어 개념
: 핵심기능 -> C언어 기능 + 공통 실행환경(놀이터), 클래스(바구니 틀), 객체(바구니)
: C언어 기능에 여러 기기에서 실행가능하도록 도와주는 공통 실행환경, 
  그리고 데이터와 메서드를 담는 클래스(바구니 틀)를 통해 객체(바구니)를 만들 수 있는 언어

- Java를 배워야 하는 이유
1) Java는 공통 실행환경이 있어서 여러 기기에서 실행 가능함
: 여러 기기의 운영 체제에서 실행할 수 있는 언어임
: 특히, 앱 개발에 적합함
2) 코드를 이해하기 쉽고, 유지보수가 쉬움
3) 안정성이 높아서 프로그래밍 오류를 방지하고 보안상 안전함
4) 대규모 앱 개발이 가능함
5) 다양한 도구와 라이브러리를 사용할 수 있음

# 스프링 기초와 원리
- 실무에서 스프링을 사용하고 있지만, 다양한 기능을 원리도 모르고 사용하는 경우가 다반사이다.
  이러한 부족한 점을 보완하기 위해 스프링 기초와 원리에 대해 공부하면서 정리하려고 한다.
- 만약 누군가가 스프링에 대해 설명해주세요. 하면 어떻게 대답할 것인가??
  아마도 외워왔던 답변대로 답을 하지 않을까? "스프링은 자바 언어 기반의 프레임워크입니다" 라고...
  그럼 자바 언어는 뭐에요?라고 한다면 "자바는 객체지향 프로그래밍 언어입니다"
  이어서 다음 질문은 뭔지 예상 갈 것이다. "객체지향이 뭐예요?"
  여기서는 말문이 막힐 것이다.
  객체지향으로 프로그램을 짜면 유지보수에 편리하다라는 것만 알고 
  객체지향 개념이 무엇인지, 왜 사용하는지에 대해서는 설명을 못할 것이다.
  또한 클래스라는 것을 사용하여 객체를 생성하고, 필요한 곳에서 사용하는 것까지는 
  코드를 구현하면서 파악을 했을 테지만 그외의 개념에 대해서는...
  이러한 부족한 점을 보완하고자 스프링 기초와 원리에 대해 공부한 것을 정리하고자 한다.
