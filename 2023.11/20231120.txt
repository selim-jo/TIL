# 트랜잭션
- 트랜잭션 명령어를 사용하다보니 트랜잭션의 개념에 대해 한번 정리해야 되겠다는 생각이 들어 아래에 정리하였다.

// 트랜잭션
- 최종 결과를 내기까지 위한 하나의 작업 단위 의미
- 오라클 DB는 개발자가 전달한 insert, update, delete문을 메모리상에서만 수행하고 디스크에 반영x
- 이는 실수로 인한 데이터 유실을 막기 위함
- DB를 조작하는 작업이 완료되고, 모두 정상적으로 되었다면 이를 디스크에 반영해야 함
- 작업이 시작되고 디스크에 반영될 때까지의 작업의 단위를 트랜잭션이라 부르며 트랜잭션이 완료되면 디스크에 반영하여 저장하게 됨

// 트랜잭션 명령어
- commit; : 트랜잭션을 완료하고 디스크에 반영함, 복구 불가
- rollback; : 트랜잭션을 취소함
- savepoint; : rollback의 단위를 지정함
