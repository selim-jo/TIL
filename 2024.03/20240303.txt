# CI, DI 개념 정리
- 인증 관련 업무를 하다보면 CI, DI 개념에 대해 접하게 된다. CI와 DI는 한국에만 있는 개인 식별 정보로 두 가지의 개념과 차이에 대해 정리해보자.

- CI (Connection Information)
: 연계 정보로 온라인 상의 개인 식별, 즉 온라인 주민등록번호라고 보면 됨
  (88 byte의 대소문자 + 숫자로 이루어진 키)
: 개인마다 고유하게 발급되는 키 값으로 서로 다른 서비스에서도 공유하여 유저를 식별할 수 있음

- DI (Duplication Information)
: 중복 확인 정보로 인증 솔루션마다 같은 개인이라도 다른 키값이 발급될 수 있음
  (66 byte의 대소문자 + 숫자로 이루어진 키)
: 서로 다른 서비스에서 공유하여 사용 불가

cf) CI와 DI는 바꾸고 싶다고 해서 개인이 바꾸지 못함
